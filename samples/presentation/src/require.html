关于jquery

  • 写代码时尽量少创建jquery对象。
  • 因为性能及功能问题，尽量使用TweenLite或者TweenMax做动画。
  • 事件侦听使用on方法，并且如果存在多个需要侦听的相同行为的元素（比如列表元素），请对它们的父级使用on方法。
  • 对于类似Ajax请求（$.get,$post,$.ajax）或类似的异步操作，避免使用回调函数。
  • Deferred Object， Q.js。

关于requirejs（一）

  • AMD解决JS相互依赖及加载问题
  • 定义一个模块，请使用 define： 
    define(['crown', 'crown.site/Scene'], function (crown, Scene) {
    var Scheme = crown.inherit('scene.BasicScene', Scene);
    return Scheme;
});
  • 使用其他模块，请使用 require： 
    require(['scripts/Main', 'domReady!'], function (Main) {
    new Main();
});

关于requirejs（二）

  • define，require 的差别在于是否需要返回一个可供其他模块使用的对象。如果代码本身的目的是想创建一个可供其他对象使用的对象，使用define；如果 只是想调用一些现有的模块实现一个功能，使用require；如果在define内部还需要动态依赖其他的模块，使用require.
  • 配置文件，paths、shim、packages
  • 压缩代码，r.js、grunt-contrib-uglify

关于继承

  • 新建类，crown.inherit(newClassName,parentClass)
  • 定义属性，crown.properties(prototype,[{name:'protoName',getter:true,setter:false}])
  • 调用父类方法, Scheme.uber.load.apply(this, arguments) 
define(['crown', 'crown.site/Scene'], function (crown, Scene) {
    var Scheme = crown.inherit("scene.BasicScene", Scene), proto = Scheme.prototype;
    crown.properties(proto, [{ name: 'allLoaded', getter: true, setter: false }]);
    proto.load = function ($content) {
        var self = this, deferred = $.Deferred();
        Scheme.uber.load.apply(this, arguments).done(function () {
            self._allLoaded = true;
            deferred.resolve();
        });
        stage.getSpinner().commit();
        return deferred.promise();
    };
    proto.enter = function (paper) {
        return Scheme.uber.enter.apply(this, arguments);
    };
    return Scheme;
});
